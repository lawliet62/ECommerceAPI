# Inventory Deduction Timing

## Context

주문 생성과 결제 완료 사이에서 재고를 언제 차감할지 결정해야 했다.

현재 주문 흐름은 다음과 같다.

1. 사용자가 장바구니에 상품을 담는다.
2. 장바구니를 기반으로 주문을 생성한다.
3. 결제가 완료되면 주문이 확정된다.

## Options

### 주문 생성 시점에 재고 차감

장점:

- 초과 판매를 막기 쉽다.
- 결제 단계로 넘어간 주문의 재고를 미리 확보할 수 있다.

단점:

- 결제 실패 시 재고 복구가 필요하다.
- 주문만 생성하고 결제하지 않은 경우 재고가 묶인다.
- 주문 만료, 재고 예약 해제 같은 추가 정책이 필요해진다.

### 결제 완료 시점에 재고 차감

장점:

- 결제 실패 시 재고 복구가 필요 없다.
- 미결제 주문 때문에 재고가 묶이지 않는다.
- 현재 프로젝트 범위에서 구현이 단순하다.

단점:

- 주문 생성 후 결제 완료 전까지 재고가 보장되지 않는다.
- 결제 완료 시점에 재고를 다시 검증해야 한다.

## Decision

현재 프로젝트에서는 **결제 완료 시점에 재고를 차감한다.**

주문 생성은 사용자의 주문 의사를 기록하는 단계로 보고, 결제 완료를 주문 확정 시점으로 본다.

## Policy

- 장바구니 추가 시점에는 상품의 active 상태와 현재 재고를 검증한다.
- 주문 생성 시점에는 장바구니가 비어 있는지, 상품이 주문 가능한 상태인지, 재고가 충분한지 다시 검증한다.
- 주문 생성 시점에는 재고를 차감하지 않는다.
- 결제 완료 시점에 주문 항목 기준으로 재고를 재검증하고 차감한다.
- 재고 차감이 성공하면 주문 상태를 `PAID`로 변경한다.

## Reason

현재 프로젝트는 재고 예약, 주문 만료, 미결제 주문 자동 취소 같은 정책을 아직 다루지 않는다.

따라서 주문 생성 시점에 재고를 차감하면 결제 실패나 미결제 주문에 대한 복구 로직이 필요해져 복잡도가 커진다.

결제 완료 시점에 차감하는 방식은 현재 학습 범위에서 더 단순하고, `Order`와 `Payment` 도메인의 책임도 명확하게 나눌 수 있다.
