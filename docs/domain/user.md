# 사용자 도메인 규칙

## 범위

사용자 도메인은 회원가입, 로그인 식별자, 암호화된 비밀번호, 권한 역할을 관리한다.

## 비즈니스 규칙

- 이메일은 필수이며 공백일 수 없다.
- 이메일은 전체 사용자 중 유일해야 한다.
- API 요청에서는 이메일 형식을 검증한다.
- 비밀번호는 필수이며 공백일 수 없다.
- API 요청에서는 비밀번호가 최소 8자 이상이어야 한다.
- 비밀번호는 저장 전에 반드시 암호화한다.
- 새로 가입한 사용자의 기본 역할은 `USER`다.
- 로그인은 이메일이 존재하고 입력 비밀번호가 저장된 암호화 비밀번호와 일치할 때만 성공한다.
- 로그인 실패 응답은 이메일 오류와 비밀번호 오류를 구분하지 않는다.

## DB 무결성 및 제약 조건

- `users.email`은 `NOT NULL`이다.
- `users.email`에는 유니크 제약을 둔다.
- `users.password`는 `NOT NULL`이다.
- `users.role`은 `NOT NULL`이다.
- `users.role`은 `USER`, `ADMIN` 값만 허용한다.

## 애플리케이션에서 보장하는 규칙

- 비밀번호 암호화는 `AuthService`에서 수행한다.
- 이메일 형식과 비밀번호 최소 길이는 요청 검증 규칙이며 DB 제약으로 강제하지 않는다.
