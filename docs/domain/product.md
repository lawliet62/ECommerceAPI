# 상품 도메인 규칙

## 범위

상품 도메인은 상품명, 가격, 재고, 활성 상태를 관리한다.

## 비즈니스 규칙

- 상품명은 필수이며 공백일 수 없다.
- 상품명은 100자 이하여야 한다.
- 상품 가격은 0보다 커야 한다.
- 상품 재고는 0 이상이어야 한다.
- 새 상품은 기본적으로 활성 상태다.
- 일반 상품 조회는 활성 상품만 노출한다.
- 관리자 상품 조회는 활성/비활성 상품을 모두 조회할 수 있다.
- 상품 정보 수정에서는 상품명과 가격을 변경할 수 있다.
- 재고 수정에서는 재고를 직접 설정할 수 있지만 0 미만으로 설정할 수 없다.
- 상품 활성화와 비활성화는 명시적인 관리자 동작이다.
- 재고 차감 수량은 0보다 커야 한다.
- 현재 재고가 차감 수량보다 적으면 재고 차감은 실패한다.

## DB 무결성 및 제약 조건

- `product.name`은 `NOT NULL`이며 100자로 제한한다.
- `product.price`는 `NOT NULL`이며 `DECIMAL(15, 2)`로 저장한다.
- `product.price`는 0보다 커야 한다.
- `product.stock`은 `NOT NULL`이다.
- `product.stock`은 0 이상이어야 한다.
- `product.active`는 `NOT NULL`이다.

## 애플리케이션에서 보장하는 규칙

- 일반 사용자에게 활성 상품만 노출하는 규칙은 Repository 조회 조건으로 보장한다.
- 재고 차감 가능 여부는 도메인 로직에서 검증한다. DB는 최종 재고가 음수가 되지 않는 것만 막는다.
- 현재 구현은 장바구니 추가 시점이나 주문 생성 시점에 재고를 예약하지 않는다.
