# 결제 도메인 규칙

## 범위

결제 도메인은 주문에 대한 결제 시도와 결제 대기 주문을 결제 완료 주문으로 전환하는 흐름을 관리한다.

## 비즈니스 규칙

- 결제는 반드시 한 주문에 속한다.
- 결제 금액은 주문 총액에서 복사한다.
- 결제 금액은 0보다 커야 한다.
- 새 결제의 기본 상태는 `PENDING`이다.
- 결제는 현재 사용자가 소유한 주문에 대해서만 생성할 수 있다.
- 결제는 주문 상태가 `PENDING_PAYMENT`일 때만 생성할 수 있다.
- 같은 주문에는 `PENDING` 상태의 결제가 최대 하나만 존재할 수 있다.
- 대기 중인 결제는 완료, 취소, 실패 처리할 수 있다.
- 결제 완료는 결제 상태를 `SUCCESS`로 변경한다.
- 결제 완료 시 주문이 여전히 결제 가능한 상태인지 다시 검증한다.
- 결제 완료 시 주문 상품 기준으로 상품 재고를 차감한다.
- 결제 완료 시 주문 상태를 `PAID`로 변경한다.
- 결제 취소는 결제 상태만 `CANCELLED`로 변경한다.
- 결제 실패는 결제 상태만 `FAILED`로 변경한다.
- 실패 또는 취소된 결제는 주문을 `PENDING_PAYMENT`로 유지하여 재결제를 허용한다.

## DB 무결성 및 제약 조건

- `payment.order_id`는 `NOT NULL`이며 `orders.id`를 참조한다.
- `payment.status`는 `NOT NULL`이다.
- `payment.status`는 `PENDING`, `SUCCESS`, `CANCELLED`, `FAILED` 값만 허용한다.
- `payment.amount`는 `NOT NULL`이며 `DECIMAL(15, 2)`로 저장한다.
- `payment.amount`는 0보다 커야 한다.
- 생성 컬럼과 유니크 인덱스로 같은 주문에 대기 중인 결제가 최대 하나만 존재하도록 강제한다.

## 애플리케이션에서 보장하는 규칙

- 결제 상태 전이 규칙은 `Payment` 애그리거트에서 보장한다.
- 주문 상태 변경과 재고 차감은 하나의 서비스 트랜잭션 안에서 수행한다.
- 결제 생성 이후 결제 금액이 주문 총액과 같은지는 DB에서 강제하지 않는다.
- 외부 결제 게이트웨이 연동은 아직 구현되어 있지 않다.
